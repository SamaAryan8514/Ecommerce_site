(*) Components in React 
--> Components are independent and reusable bits of code. They serve the same purpose as JavaScript functions, but work in isolation and return HTML.
--> They are basically a piece of the website.
--> We can create our own HTML elements using Components
--> Basically, we use two types of Components, i.e, Class components and Function components.
**--> components makes a particular section of code reusable, by providing it with a standard function component name, i.e, the 
piece (or component) can be used for several no. of times by just calling the self-closing component of the functional component.

(*) Usecase of Components :- Basically, while building a website, it is easier to implement the functioning of the website
and to improve the flexibility of the website by splitting the website into smaller pieces(Components), i.e, by splitting
the website into smaller pieces(or components), we can work on a small piece of the website at a time.

--> So, basically, the pieces or parts in which the website has been splitted into, are referred to as --> "components".

(*)State
--> React components has a built-in state object, which is the Component's own data that can change and cause the UI to re-render.
--> When a state changes, React re-renders the component automatically.
--> Each component can mainatins its own state.
--> State is used for intertactive or dynamic content.
--> It is basically a data that changes over time, and which is connected to the HTML.

--> An array which declares a state in React has :-  (1) arr[0](first element)--> representing the current value present during
rendering of the UI and (2) arr[1](second element)--> representing the updater function(which is responsible for the updates to

(*)Updater function

HTML).

(*) Lifting the state up --> it means to share the state between the components of a HTML(webpage), by interaction procedure via

(*)Hooks
--> These are the special functions that let us use or insert React features(state,lifecycle,etc.) in functional components.
---> Common Hooks :
(1) useState
(2) useEffect
(3) useContext
(4) useRef

(*)useEffect hook
--> it runs some code(defined or written inside it) after a particular component is created or updated.
--> it is a React hook used to perform side effects such as data fetching, subscriptions, or DOM updates in functional components.
--> it runs after the components renders(i.e, created or updated), ensuring the UI is updated before the effect executes.
--> it consists of a dependency array, which controls when the effect runs(on every render, once, or when specific values change).
--> it can return a cleanup function to remove side effects like event listeners or timers when the component unmounts or updates.

--> providing '[]' as dependency array tells implies  useEffect to only run once.

(*)useRef hook
---> it basically saves an HTML element from the particular component. 
---> it is a React hook used to create a mutable reference that persists across component re-renders.
---> it is commonly used to directly access and manipulate DOM elements without causing re-rendering.
---> Updating a "useRef" value does not trigger a re-render, unlike state updates.
---> it is also used to store values(like timers or previous state) that need to persist between renders.

(**)Routing in React
--> Routing in react lets us create multiple pages uisng 1 HTML file, which helps us reuse our HTML code.
--> The process of navigating between different components based on the URL without reloading the page.
--> Routing in React is used to navigate between different pages or views in a single-page application (SPA).
--> React uses client-side routing, meaning the page does not reload when navigating.
--> Routing is commonly implemented using React Router.
--> Routes are defined using the <Route> component.
--> The <BrowserRouter> component wraps the application to enable routing.
--> Each route maps a URL path to a specific component.
--> The <Link> component is used for navigation instead of <a> tags.

(*)<Route> component in React
--> The <Route> component is used to define a mapping between a URL path and the React component that should be rendered when that  path is visited.
--> It is used inside "<Routes>" component to handle the navigation between various web pages.

(*)<Routes> component
--> The "<Routes>" component is used to group multiple <Route> components and ensures that only the matching route is rendered.
--> It is a kind of container for all "Route" definitions.

(*)<BrowserRouter> component
--> The <BrowserRouter> component wraps the entire React application and enables routing using the browserâ€™s URL and history API.
--> It basically provides Routing capability to the React app.

(*)<Link> component
--> The <Link> component is used to navigate between routes without reloading the page.
--> They are basically placed, in place of "<a>" tag  to provide smooth and flexible navigation, without reloading the page,
    using JS to do so.
--> Generally, "<Link>" uses an attribute "to" in place of "href" to carry its workflow properly.
